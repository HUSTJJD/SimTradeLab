## 0. TL;DR（三段话总结）

1. **思路本质**：在「主营产品具有外部可观测现货价格」的优质周期股里，用「商品价格 + 估值分位」识别周期底部，**底仓 + 网格** 组合持仓，用「网格清空」与「盈利状态下涨停打开」两条规则离场。
2. **核心评估**：思路**整体可行且具备真实 alpha 来源**（产品价格→盈利→股价的产业传导链是公认有效的），但**关键风险**集中在三处：(a) 周期底部识别——容易"抄底抄到半山腰"；(b) 单边下跌时网格满仓套牢；(c) "涨停打开就卖" 在周期主升浪中可能过早离场。
3. **本文给出的优化方向**：在选股层加 **财务质量过滤** 与 **价格-股价相关性筛选**；在建仓层用 **多因子底部打分 + 分批建仓 + 硬止损** 替代"一次买 1/2 底仓"；在网格层引入 **动态步长 / 不对称网格 / 越跌越多的资金阶梯 / 网格仓位上限**；在退出层把模糊条件**精确化为可计算的判据**，并增加"周期顶部 + 高水位回撤"两道兜底。

## 1. 原始思路整理

### 1.1 用户原始描述（直译）

| 环节 | 用户描述 |
|---|---|
| 选股 | 自动选优质周期股；类似 兴发集团、云天化、杭氧股份、新和成；**主营产品在其他平台能查到实时价格**便于监控 |
| 建仓 | 在**周期底部**时，买入 **1/2 底仓** |
| 网格 | 启动网格交易，**每格 3%** |
| 退出 a | **网格卖完之后**就退出，短期不再买入 |
| 退出 b | 当**本只股票处于盈利状态**，并且**涨停**，并且**打开涨停**时就卖出 |

### 1.2 关键意图归纳

我把上面的描述拆成更结构化的目标和子模块：

- **选股目标**：找一类"基本面有锚、波动够大、有产业逻辑"的标的——既有**周期回归**带来的网格震荡收益，又有**主营产品涨价**带来的趋势收益。
- **建仓目标**：用低位"半仓底仓"承接长波趋势收益，把另外一半留给网格运转——这是**底仓 + 网格** 而不是纯网格的关键差别。
- **退出目标**：避免两类亏损：(a) 周期反转后的"满仓套死"，(b) 主升浪结束后的"利润回吐"。所以**退出 a** 防"再被套"，**退出 b** 防"利润回吐"。

### 1.3 用户给的四只标的及其"可监控产品"对照

> 这是策略能否落地的核心：**有没有真实可拿到的产品价格序列**。

| 股票 | 主营产品 | 公开/半公开价格来源 |
|---|---|---|
| 兴发集团（600141） | 草甘膦、磷矿石、有机硅、黄磷 | 百川盈孚（草甘膦/有机硅日报）、隆众资讯、卓创资讯、化工 707 |
| 云天化（600096） | 磷酸一铵 (MAP)、尿素、复合肥、磷矿 | 中国化肥网、卓创资讯（化肥日报）、Mysteel 农产品、生意社 |
| 杭氧股份（002430） | 工业气体（液氧、液氮、液氩）、空分设备 | 卓创资讯（工业气体）、SMM 部分指数；**注意：工气价格区域性极强，公开度最差** |
| 新和成（002001） | 维生素 A/E、蛋氨酸、香精香料、PPS | 博亚和讯（饲料/维生素）、卓创资讯、中国饲料行业信息网 |

## 2. 可行性分析（按环节逐项打分）

> **打分约定**：✅ 可行 / ⚠ 有条件可行 / ❌ 难以落地。括号里给出主要风险点。

### 2.1 选股层 ⚠（数据获取难度 + 主营定义模糊）

**值得做的理由**：
- "主营产品价格→营收→利润→股价" 的传导链在化工、化肥、有色、农产品板块**长期有效**——这是产业派 / 周期派研究员的核心 alpha 之一。
- 优质周期股（高 ROE 中位数、低负债率、稳定经营现金流）**穿越周期能力强**，相对垃圾周期股回撤更可控。
- 把"商品价格"作为选股锚定，等于把**基本面信号**和**网格的技术面**叠加，比纯波动率选股稳健。

**主要难点**：
- **商品价格数据**：高质量来源（卓创、隆众、Mysteel、博亚和讯）多为付费 API；免费抓取有反爬与合规风险；数据频率多为日级，不足以做日内策略。
- **主营产品权重模糊**：兴发集团既有草甘膦、又有磷矿、又有有机硅，**到底用哪个价格、还是加权？** 加权用什么权重？营收占比？毛利占比？需要额外建模。
- **周期股 ≠ 优质公司**：单纯按"周期股"筛会混入瑞贝卡、某些高负债磷化工等"伪周期"——必须叠加财务质量过滤（见 §3.1）。
- **数量 vs 质量**：仅 4 只样本太少（不能分散行业风险）；扩到 30+ 只又面临数据维护成本。

### 2.2 周期底部识别 ⚠（最难的一环，"接飞刀"风险）

**值得做的理由**：
- 周期股的 PB 分位、产品价格分位，在历史回测中确实呈"低分位买入、跨周期持有"显著正收益（参考北大光华、申万宏源等多篇周期股研究）。
- 商品价格序列比股价"领先 / 同步"——若用产品价格分位作为额外信号，相当于**用产业数据校准择时**。

**主要难点**：
- **底部 ≠ 最低点**：分位数 < 20% 可能跌到 < 10%，"底部"区间可能持续 12–24 个月（典型如 2014–2015 草甘膦、2020 工业气体）。**"一次买 1/2"的左侧抄底极易承受 20–40% 浮亏**。
- **数据可得性**：PB / PE 分位、产品价格分位需要 **5–10 年** 历史数据才有统计意义；新上市的周期股没有完整周期可参考。
- **行业差异巨大**：磷化工的周期约 2–4 年，工业气体约 3–7 年，养殖业（猪）约 3–4 年，需要**因行业差异化设定窗口**。
- **结构性变化无法用历史**：例如 2015 后环保去产能、双碳政策——历史分位数失效。

### 2.3 网格层（3% 固定步长） ⚠（思路 ok，参数过粗糙）

**值得做的理由**：
- 周期股波动大（年化 30%–50%）+ 均值回归性较强，**理论上是网格的好土壤**。
- "底仓 + 网格"组合（用户已隐含）比纯网格更稳健：底仓吃趋势收益，网格吃震荡收益。

**主要难点**：
- **3% 固定步长在不同价位、不同波动率下行为差异极大**：
  - 低价股（10–20 元）的 3% ≈ 0.3–0.6 元，极易触发；
  - 高价股（80–150 元）的 3% ≈ 2.4–4.5 元，可能一周才一次。
  - 低波动期触发太频繁，高波动期穿越太多格无法"赚到"。
- **单边下跌时满仓**：例如 2018 兴发集团从 12 元跌到 4 元（-67%）。如果一开始 1/2 底仓 + 等距网格资金，**很可能在跌到 6–7 元时已耗尽全部资金，剩下的 4–5 元区间只能干瞪眼**。
- **网格触发不等于盈利**：每次买卖间隔需要扣手续费（双向约 0.1%–0.2% + 印花税 0.1% = **每次 0.2–0.3% 摩擦**），3% 网格的实际单格毛利约 2.7–2.8%。如果穿越成本高（如多次穿越同一价位），收益会比模型测算少 30%+。
- **现有 `grid_mining/miner.py` 的网格模拟有一处隐患**：`grid_profit += (price - last_grid_price)` 假设每次只持 1 单位且**忽略买入成本与税费**，会**系统性高估**网格收益（详见代码 `calculate_grid_score()`）。

### 2.4 退出 a："网格卖完后退出，短期不再买入" ⚠（语义模糊）

**值得做的理由**：
- 这是个非常正确的"风险闸门"思路：一旦本轮震荡区间被向上突破（网格全部卖完），意味着趋势可能切换，应该**让趋势走完再考虑下一轮建仓**。

**需要厘清的语义**：
- **"卖完"指什么**？三种可能解读：
  1. 网格累计卖出股数 = 网格累计买入股数（即网格持仓归零，**底仓不动**）
  2. 网格 + 底仓全部卖光
  3. 仅指"已经触发了 N 次卖出"达到上限
- **"短期"是多长**？2 周？1 个月？6 个月？需要客观条件而非感觉。

**主要风险**：
- 如果"卖完"=网格部分清零、底仓保留，则**底仓的退出何时？** 必须由退出 b 接力，但退出 b 触发条件苛刻，可能长期持有底仓。
- 如果"短期不买入"用日历时间（如 30 天），可能错过同一周期内的二次抄底机会；如果用价格条件（如再跌 X%），又面临阈值如何选。

### 2.5 退出 b："盈利 + 涨停 + 涨停打开 → 卖出" ⚠（思路对，参数极易错杀主升浪）

**值得做的理由**：
- "涨停打开" 是 A 股经典情绪信号——多头情绪短期透支，**短线确实容易回撤**。
- 配合"持仓盈利"前提，避免亏损状态下错卖。

**主要风险**：
- **周期股主升浪中的涨停极常见**——一个完整主升浪可能包含 5–10 个涨停板，**初期就因"涨停打开"卖出会错过 50–100% 涨幅**（兴发集团 2021 主升浪就有数次涨停打开后再度封板）。
- **"打开涨停"无客观定义**：触及涨停后回落多少算"打开"？回落 0.5%、1%、2%？尾盘再度封板算不算"未打开"？
- **"盈利状态"基于哪个成本**：底仓成本、移动加权成本、个股累计盈亏？不同口径触发时点可能差几个月。
- **流动性陷阱**：小市值周期股涨停打开时如果想卖，可能因为下一秒重新封板、或瞬间砸跌停，**实际成交不一定能在打开点完成**。

## 3. 优化建议（按环节逐项展开）

### 3.1 选股层优化

#### A. 增加财务质量过滤（避免周期陷阱）

| 指标 | 建议阈值 | 说明 |
|---|---|---|
| 资产负债率 | < 60% | 高负债周期股在产品价格下行期最危险（如某些磷化工高杠杆标的） |
| 经营活动现金流 / 净利润 | > 80%（近 3 年平均） | 排除"账面盈利、现金流不行"的标的 |
| ROE 行业分位 | 近 5 年 ROE 中位数 ≥ 行业 60% 分位 | 穿越周期的盈利能力 |
| 商誉 / 净资产 | < 30% | 减少并购雷 |
| 经营现金流连续为正 | 近 5 年至少 4 年为正 | 验证主业造血能力 |
| 实控人 / 高管股权质押率 | < 30% | 排除质押爆仓风险 |

> 实施建议：在 `SimTradeLab` 的 fundamentals/valuation 数据上加一层 quality filter，作为 `grid_mining/miner.py` 的前置过滤器。

#### B. 商品价格 — 股价相关性筛选

- **目的**：确保"商品价格 → 股价" 的传导机制确实存在，不是讲故事。
- **方法**：在选股阶段计算**滚动相关性**：
  - 取标的过去 3 年周线收益率，与"加权主营产品价格"周线变化做相关性。
  - 主营产品多于一种时，按**营收占比**或**毛利占比**加权（数据取自年报"主营业务构成"）。
  - **筛选标准**：相关性 ≥ 0.5（强相关）；< 0.3 的剔除。
- **替代方案**（数据缺失时）：用 **行业指数代理**——如申万化工指数、Wind 工业气体指数——比股价对个股先行 1–2 个月。

#### C. 候选池建议（扩展示例）

为了分散单一行业风险，建议候选池覆盖 **5–7 个细分行业、20–30 只标的**：

| 板块 | 标的示例 | 锚定产品 |
|---|---|---|
| 磷化工 | 兴发集团、云天化、川发龙蟒、湖北宜化、川恒股份 | 草甘膦、磷酸一铵、磷矿石 |
| 工业气体 | 杭氧股份、华特气体、金宏气体 | 液氧、液氮、电子特气 |
| 维生素 / 营养品 | 新和成、浙江医药、安迪苏、花园生物 | VA、VE、蛋氨酸、维生素 D3 |
| 钛白粉 | 龙佰集团、中核钛白 | 钛白粉、钛精矿 |
| 制冷剂 | 巨化股份、三美股份、东岳集团 | R32、R134a 等氟化工 |
| 有色金属 | 紫金矿业、神火股份、洛阳钼业、华友钴业 | 黄金、铝、铜、钴 |
| 农药 | 扬农化工、利尔化学、广信股份 | 麦草畏、草铵膦、敌草快 |
| 农产品养殖 | 牧原股份、温氏股份、海大集团 | 生猪、玉米 / 豆粕 |

### 3.2 建仓层优化

#### A. 周期底部多因子打分（替代"主观判断"）

```
底部综合分 = 0.30 × s(主营产品价格 5 年逆向分位)
          + 0.20 × s(PB 5 年逆向分位)
          + 0.20 × s(距前高回撤幅度)
          + 0.15 × s(经营现金流 / 市值)
          + 0.15 × s(股价 250 日 RSI 逆向)
```
- 各项 `s(·)` 标准化到 [0, 100]。
- **打分 ≥ 70** 视为"底部信号成立"，开始建仓；70–80 中等，> 80 强信号。
- **打分需要持续性**：建议要求**连续 5 个交易日**得分 ≥ 70 才触发，避免单日噪声。

> 这把"周期底部"从主观判断变成可计算、可回测、可调参的客观信号。

#### B. 分批建仓（替代"一次买 1/2 底仓"）

| 阶段 | 触发条件 | 加仓比例（占目标底仓） |
|---|---|---|
| T0 | 底部信号成立 | 30% |
| T1 | 价格再下跌 ≥ 8% 或 底部分数再升 ≥ 5 | 30% |
| T2 | 价格再下跌 ≥ 8% 或 30 日内未触及 T0 价位（价格右侧反弹确认） | 40% |

> 优于"一次买 1/2"：左侧分批承接 + 右侧确认补齐，**对左侧失败有韧性**，对右侧错过有补偿。

#### C. 硬止损线（周期股的生死线）

满足以下任意一条触发**全部清仓**（含底仓 + 网格）：
- 主营产品价格在初次建仓后再下跌 **≥ 30%**，且持仓累计浮亏 ≥ 25%；
- 公司发生重大基本面恶化（停产事故、安全事故、被立案、ROE 转负 2 个季度）；
- 行业出现**结构性替代**（如新技术、政策强制淘汰）。

> 没有止损的周期股策略不是策略，是信仰。

### 3.3 网格层优化

#### A. 动态网格步长（替代固定 3%）

- 步长 = `max(2%, min(5%, k × ATR(20) / 当前价格))`，`k` 默认取 1.5。
- 高波动期自动放宽到 4–5%，低波动期收紧到 2–2.5%；避免"频繁穿越-高摩擦"和"长时间无信号"。

#### B. 不对称网格（迎合周期反转主升浪）

- **跌买网格更密**：2.5% 触发买入；
- **涨卖网格更稀**：3.5%–4% 才触发卖出；
- **逻辑**：周期底部多吸筹（密买），主升浪少卖出（稀卖），让趋势奖励更多留下。

#### C. 越跌越多的资金阶梯（"金字塔加仓"）

| 距建仓基准价 | 单格买入资金倍数 |
|---|---|
| -3% | 1× |
| -6% | 1.5× |
| -9% | 2× |
| -12% | 2.5× |
| -15% 及以下 | 3×（且达上限后停止） |

> 越接近"理论底底部"，权重越高，符合"机会越好越下重注"。

#### D. 网格仓位上限（保护性）

- 网格 + 底仓合计**不超过单股资金的 100%**（即不加杠杆）；
- 网格部分**不超过该股可用资金的 50%**（剩余 50% 为底仓 + 极端补仓预备金）；
- 整个策略组合中**单只标的占比 ≤ 15%**，单一行业占比 ≤ 30%。

### 3.4 退出层优化

#### A. 退出规则 a 的精确化

- **"卖完"明确定义**：网格累计卖出股数 ≥ 网格累计买入股数 + 初始网格预算（即"网格部分仓位归零"）。**底仓不卖**，由规则 b 处理。
- **"短期不买入"客观条件**：满足以下任一则解除冷却、可重新进入：
  - 价格回到本轮高点 -25% 且**底部综合分 ≥ 70**；
  - 或者：自网格清空起经过 ≥ 60 个交易日且 **底部综合分 ≥ 65**；
- **冷却期内允许的动作**：仅限"持有底仓 + 监控信号"，不开新仓、不补网格。

#### B. 退出规则 b 的精确化（避免错杀主升浪）

仅在以下**全部满足**时触发：

1. **整体浮盈条件**：底仓 + 网格综合浮盈 ≥ **30%**（避免小盈即跑）；
2. **涨停定义**：当日收盘价 / 前一日收盘价 ≥ 1.0998（≈ 10%，主板）或 ≥ 1.1998（创业板/科创板的 20%）；
3. **涨停打开定义**：日内**触及涨停价**后，价格曾**回落 ≥ 1.5%**，且**当日收盘未再封板**；
4. **周期顶部联动信号**（推荐增加，二选一）：
   - 主营产品价格 5 年分位 ≥ **80%**；
   - 或者 PB 5 年分位 ≥ **70%**；

**卖出比例分级**：
- 仅满足 1+2+3：**卖出底仓 50%**（保留另一半博主升浪续航）；
- 同时满足 1+2+3+4：**卖出全部底仓 + 网格剩余仓位**（认为是阶段顶）。

#### C. 增加"高水位回撤止盈"（兜底）

- 浮盈 ≥ **50%** 后，启用"高水位回撤"机制：
- 浮盈从最高水位回撤 ≥ **15%** → 强制减仓 50%；
- 回撤 ≥ **25%** → 全部清仓。

> 防止"涨停打开 b 不触发"但行情已掉头的情况。

#### D. 时间退出（避免长期套牢）

- 持仓 ≥ **3 年**且未触发任何退出规则：强制人工评估。
- 若行业基本面发生质变（如新技术替代、政策淘汰），止损出清。

## 4. 风险提示（必须正视）

| 风险 | 描述 | 缓解措施 |
|---|---|---|
| **周期"伪底"** | 抄底后继续下跌 30%–50% | 多因子打分 + 分批建仓 + 硬止损 |
| **网格满仓套牢** | 单边下跌耗尽资金后无法继续运转 | 网格仓位上限 + 资金阶梯 + 极端备用金 |
| **基本面暴雷** | 安全事故 / 政策淘汰 / 财务造假 | 财务质量过滤 + 主动监控公告 + 硬止损 |
| **流动性风险** | 小市值股一字板出不来 | 候选池剔除日均成交 < 5000 万元的标的 |
| **商品价格-股价短期脱钩** | 情绪 / 资金面驱动股价偏离基本面 | 仅用于选股 + 底部信号，不用于日内 |
| **数据源失效** | 抓取被封 / 数据延迟 / 数据错误 | 多源备份 + 人工抽检 + 数据完整性校验 |
| **过度拟合** | 参数对历史数据贴合过度 | 样本外回测 + 滚动窗口验证 + 蒙特卡洛 |
| **系统性风险** | 大盘暴跌时所有标的同跌 | 组合层仓位上限 + 极端市况减仓规则 |
| **"涨停打开"误判** | 尾盘再次封板 / 一字板未打开假信号 | 用收盘是否封板 + 当日成交量异常作为佐证 |
| **税费 / 摩擦超预期** | 高频网格交易摩擦累积 | 网格步长 ≥ 2%；动态步长根据波动率调节 |

## 7. 结论

您这套思路的**底层逻辑成立**——把"周期股的产业逻辑"作为大方向、把"网格"作为具体战术，是一个**alpha + beta 兼顾**的好方向。但当前描述更接近"投资思想"而非"可执行规则"，需要在以下三处尤其加固：

1. **底部识别**——从主观感觉变成多因子打分；
2. **网格机制**——从固定 3% 变成动态、不对称、有资金阶梯、有仓位上限；
3. **退出规则**——把"短期不买入"和"涨停打开"精确化为可计算的客观判据，并增加"周期顶部 + 高水位回撤"两道兜底。
